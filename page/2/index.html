PyramidMap Javascript API►04：Layer control

PyramidMap Javascript API Layers control

In essence, layers follow the map manufacturer’s own specifications or OGC standard specifications, with built-in data structure of attribute data, spatial data, coordinate system definition and symbol graphics rendering. Therefore, GIS is a processing system for spatial data distribution and graphics rendering, which can correctly express and transfer the meaning of map data, and complete the specific map functions. PyramidMap is oriented to map application layer, and provides solutions in Web GIS and python data processing.

PyramidMap supports various types of vector and image layer display, and transforms, adjusts, appends and merges different coordinate systems to help users construct their own map data and display system through the built-in JS component.

The built-in graphics rendering function of PyramidMap provides a high contrast color system. Layer control system can construct hierarchical structure and classification according to element type and business type, and perfectly present map features at each level.

PyramidMap provides vector layer,raster layer suchas terrain map and satellite map and other multi style map layers, and can do complex rendering for them. Only the most important point information that meets the user’s filtering criteria and zooming range can be displayed.

01：What's the PyramidMap?

What's the PyramidMap?

PyramidMap is a suite components for map developping in JavaScript and python

PyramidMap is an Intelligent map component developed by pyramidMap studio which provide two product lines in web and python providing professional map services for users, in order to build a practical map application system quickly. PyramiadMap covers JavaScript API library for Web GIS applications and python map tools , from desktop mapping to GIS Server engine and web development components, supporting multi terminal and cross platform deployment. Pyramidmap engine implements elastic function deployment through plug-in mode, fully supports 2D and 3D map application based on webgl, supports WMS / WFS / WCS / KML and other standard interfaces following OGC specification therefore realizes data communication and integration between heterogeneous GIS platforms. Pyramidmap engine has built-in support for ArcGIS, GeoServer, openlayer, leaflet, cesium, Google map, Mapbox, and Tianditu/Amap / Baidu map, and provides Python based interface to meet the all-round data processing needs of map users.

In the pyreamidmap, map design is around map application, and the two are complementary. The design of layer data structure is In the service of the map application, and the purpose is to meet the practical and humanized operation of data input and output and UI interaction to the greatest extent.

PyramidMap Javascript API►03：Base Functions

PyramidMap Javascript API base functions

Vector and grid

Support various vector and raster maps, including ArcGIS Online, Tiantu, Mapbox and AMap / Baidu map support topographic map and satellite map, and coordinate system conversion can be carried out according to actual needs. Pyramidmap supports terrain rendering based on terrain data in the 3D environment .

Offline support

Support offline deployment, calling online map resources or hosting in GIS Server. All kinds of offline map services on server. Users can deploy business layers on ArcGIS Server and GeoServer, call and load various map functions through API interface provided by pyramidmap, such as attribute query, spatial query, layer management, data import and export, statistical report, etc.

Symbol map symbolization management

Dynamic real-time change map appearance, personalized icon and color settings, dynamic switching thematic symbols, support preset and interface settings.

3D map

Support ArcGIS, cesium, mapbox, Gaode and other 3D maps, fully compatible with BIM / CIM and other building layers, realize the organic integration of true 3D and true scene

Global data

It covers all kinds of global map data and coordinate systems, including vector, raster, satellite, remote sensing image, etc., and supports WGS84 and web
All standard space and projection coordinate systems such as Mercator, CGCS2000, Xian80 and Beijing 54 can be transformed and adjusted according to the production environment.

Support for third party applications

Support three party system docking, such as SCADA data acquisition software, automatic chart generation software, BI data analysis software. Pyramidm
ap provides standard API interface, integrates map and production data, and forms rich thematic maps through data analysis and summary.
