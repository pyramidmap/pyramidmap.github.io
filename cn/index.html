PyramidMap API For JS 简介

PyramidMap是金子塔底图工作室面向敏捷性应用与开发的地图产品。秉承专业化技术、敏捷化部署、实用化功能三原则，向用户提供完整的地图应用体系。基于pyramidMap地图平台，用户可以快速构建自己的地图产品，从简单的桌面端工具到web端JS API，整个应用体系覆盖Gis Server服务端引擎及其对应的Client客户端组件，支持多终端、跨平台部署。PyramidMap引擎通过plugin插件模式实现弹性功能部署，全面支持基于WebGL的2D及3D地图应用，遵循OGC规范，支持WMS/WFS/WCS/KML等标准接口，实现异构GIS平台间的数据通讯及整合。PyramidMap地图引擎以widget组件模式输出地图服务及开发接口，内置对ArcGIS、Geoserver、OpenLayer、Leaflet、Cesium、Google Map、Mapbox、高德/百度地图的支持，并提供基于python的接口，满足地图用户全方位的数据处理需求。下面是PyramidMap实现一幅地图的逻辑框架图：

PyramidMap API For JS 接口►基本功能

PyramidMap API For JS 基本功能

矢量及栅格

支持各种矢量及栅格地图，包括：ArcGIS online、天地图、Mapbox、高德/百度地图，支持地形图和卫星图，并可根据实际需要进行坐标系转换。在三维环境中支持根据Terrain地形数据进行3D地貌呈现。

支持离线

支持离线部署，调用在线地图资源或托管在GIS Server上的各种离线地图服务。用户可将业务图层部署在ArcGIS、GeoServer上，通过PyramidMap提供的API接口，调用加载并实现各种地图功能，如属性查询、空间查询、图层管理、数据导入导出、统计报表等。

Symbol地图符号化管理

动态实时改变地图的外观，个性化图标和颜色设定，动态切换专题符号，支持预设定及接口设定。

3D地图

支持ArcGIS、Cesium、MapBox、高德等多种三维地图，完全兼容BIM/CIM等建筑物图层，实现真三维、真场景的有机融合。

全球数据

覆盖全球各种类型和坐标系统的地图数据，包括矢量、栅格、卫星、遥感影像等，支持WGS84、web墨卡托、CGCS2000、Xian80、Beijing54等所有标准空间和投影坐标系统，可根据生产环境进行转换调整。

支持第三方应用

支持三方系统对接，如SCADA数据采集软件、自动图表生成软件，BI数据分析类软件。PyramidMap提供标准API接口，将地图和生产数据进行整合，通过数据分析汇总，形成丰富的专题地图。

PyramidMap 地理数据处理工作流►6：栅格处理

PyramidMap 地理数据处理工作流平台栅格处理

影像数据处理：

包括影像压缩、影像切分、影像合并、影像转换、无效值处理和构建影像金字塔。说明如下：

5.1：影像压缩：

压缩参数的设定可以按照程序界面引导选择，构建金字塔可对影像进行分层索引，这点对于提高大数据影像的加载显示速度非常有效。影像中的无效值会造成黑色背景，选择无效值处理选项，在压缩过程中会自动对无效像素进行过滤，从而提高影像质量，简化处理流程。

处理过程会保存日志，处理完毕进行消息提示。

5.2：影像切分：

切分数量通过目标区的切分基数设定，规则是长宽各按选择的数量切分，最终的切片数为n*n。过程中，可以选择是否对切分后的影像进行无效值处理，从而避免黑色无效像素的出现。如果切分的源影像本身已经过滤了无效值，则可以保持现状，不作处理，这要取决于预先明确源数据无效值状态。

上图就是经过切分处理的切片与原图像的对比。

5.3：影像合并：

影像合并与切分是逆过程，程序可以将属于同一坐标系的等分切片合成为一张完整的影像。

目标区参数控制合并规则，可以通过速度与精度的适配，选择适合的合并模式。可以对合并后的影像进行无效值处理，从而避免黑色无效像素的出现。如果合并的所有源影像已经过滤了无效值，则可以保持现状，不作处理，这要取决于预先明确元数据状态。构建金字塔，有助于提高影像的加载速度，可以选择执行。

上图就是切片影像与合并图像的对比。

5.4：影像类数据转换：

完成影像类地图在不同数据类型、像素位深、色彩映射表及波段、多维数据处理等之间进行相互转换。

您可以设置目标的像素深度类型和格式，也可以与原文件保持不变。正常情况下，文件类型栅格数据集1位栅格包含8位金字塔层，而在地理数据库中只包含1位金字塔层，这使得显示看起来很不美观。因此，在栅格导入到地理数据库时，您可以选择将输入的1位光栅数据集转换为8位光栅数据集。

如果栅格输入具有颜色映射表，则可以将目标转换为三波段光栅。这对于包含不同颜色映射表的镶嵌数据集光栅非常有用。还可以设置光栅或镶嵌数据集的像素位深度，如果未定义，则默认从第一个光栅数据集获取。

如果目标光栅的像素类型与输入不同（例如，从16位到8位），可以选择将值缩放到新范围；否则，将丢弃不符合新像素范围的值。

如果要抑制从屏幕捕捉或扫描地图或扫描文档的中的噪声，可以选择使用“颜色映射表”将8位3波段（RGB）光栅转换为单波段光栅数据集。但对于卫星、航空影像或专题栅格数据，不建议如此操作。

与矢量图层坐标系转换相同，具有标准坐标系定义的影像类，可以在不同坐标系之间相互转换，程序提供转换流程处理，实现批量化工单。以下是操作流程：

5.6：批量构建影像金字塔：

对影像构建金字塔可以加快影像加载速度。程序支持影像金字塔的参数化批量生成。

金字塔可以加速图像加载。程序支持图像金字塔的参数化批量生成。通常情况下，在光栅上构建金字塔可以改善显示效果表演。只构建一次，则后期每次显示都会访问金字塔。对于行或列中小于1024像素的光栅，即使您选择，也不会构建金字塔。小波变换压缩光栅（如ECW和MrSID）不需要构建金字塔，因为在编码时已经创建了内部金字塔。在构建栅格金字塔过程中，您可以指定金字塔等级（可选）、跳过等级（可选），重采样方式（可选），压缩类型（可选），压缩质量（可选），所有这些参数都在上面列表中提供说明。
