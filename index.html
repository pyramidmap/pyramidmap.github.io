PyramidMap GeoProcess workflow►6：Raster processing

PyramidMap GeoProcess workflow platform raster processing

Raster processing

It includes raster compression, raster slice, raster merge, raster conversion,
noDataValue set and raster pyramid construction. The explanation is as follows:

1:Raster compress：

The setting of compression parameters can be guided and selected according to the program interface, and the pyramid can be constructed to index the images hierarchically, which is very effective for improving the loading and display speed of big data images. The invalid value in the image will cause black background. Selecting the invalid value processing option will automatically filter the invalid pixels in the compression process, so as to improve the image quality and simplify the processing flow. The log will be saved during processing, and a message will be prompted after processing.

2:Raster slice:

The slice number is set by the segmentation base of the target area. The rule is that the length and width are divided according to the selected number, and the final number of slices is n * n. In the process, you can choose whether to process the invalid value of the segmented image, so as to avoid the appearance of black invalid pixels. If the segmented source image itself has filtered the invalid value, it can keep the status quo and do not deal with it, which depends on clarifying the invalid value state of the source data in advance.

The above image is the contrast between the sliced image and the original image.

3:Raster merge

Merging and slice is a reverse process. The program can synthesize a complete
image in the same coordinate system.

The merging rules are controlled by the parameters of the target area, and the suitable merging mode can be selected by the matching of speed and precision. It can deal with the invalid value of the merged image, so as to avoid the appearance of black invalid pixels. If all the merged source images have been filtered with invalid values, they can remain as they are and not be processed, depending on the metadata state. The construction of the pyramid helps to improve the loading speed of the image, which can be executed selectively.

4: Raster conversion:

The raster is transformed among different data types, pixel depth, color map
table, band and multi-dimensional data processing.

You can set the pixel depth type and format of the target or remain unchanged.Normally, 1-bit raster data set contains 8-bit pyramid layer in the file system, but only contain 1-bit pyramid layer in the geographic database, which makes the display look unattractive. Then You can Choose whether to
convert the 1-bit raster dataset input to an 8-bit raster dataset when being imported to a geo database. If the raster input has a color map table, you can convert the target to a three bands raster. This is useful for mosaics rasters which contain different color mapping tables.And you can set the pixel bit depth of a raster or a mosaic dataset, this default value is obtained from the first raster dataset if not defined. If your target raster’s pixel type is different from the input (for example,from 16 to 8 bits), you can choose to scale the value to the new range; otherwise, the value will be discarded which does not conform to the new pixel range. If you want to suppress the noise in the rasters which are captured from screen or scanned map or scanned document, you can select converting 8-bit 3-band (RGB) raster into a single band raster data set with color mapping table.However, it is not recommended to use it for satellite, aerial image or thematic raster data.

The image class with standard coordinate system definition is the same as the transformation of vector layer coordinate system. It can be converted between different coordinate systems. 6: Raster pyramid building:

The pyramid can speed up the image loading. Program supports the parametric batch generation of image pyramid.

In general, building a pyramid on raster can improve the display performance.Just building once, and the pyramid is accessed each time the raster
displayed. The raster with less than 1024 pixels in rows or columns, no pyramid will be built even if you choose to do.Wavelet compressed raster (such as ECW and MrSID) do not need to build pyramids also because having an internal pyramid created when encoding. In the pyramid building, you can design the pyramid_level(optional),the skip_first(optional),the resample_technique(optional),the compression_type(optional),the compression_quality(optional),
the skip_existing(optional), all of these are indentified in the parameters list above.

PyramidMap API For JS►Base Functions

PyramidMap API For JS base functions

Vector and grid

Support various vector and raster maps, including ArcGIS Online, Tiantu, Mapbox and AMap / Baidu map support topographic map and satellite map, and coordinate system conversion can be carried out according to actual needs. Pyramidmap supports terrain rendering based on terrain data in the 3D environment .

Offline support

Support offline deployment, calling online map resources or hosting in GIS Server. All kinds of offline map services on server. Users can deploy business layers on ArcGIS Server and GeoServer, call and load various map functions through API interface provided by pyramidmap, such as attribute query, spatial query, layer management, data import and export, statistical report, etc.

Symbol map symbolization management

Dynamic real-time change map appearance, personalized icon and color settings, dynamic switching thematic symbols, support preset and interface settings.

3D map

Support ArcGIS, cesium, mapbox, Gaode and other 3D maps, fully compatible with BIM / CIM and other building layers, realize the organic integration of true 3D and true scene

Global data

It covers all kinds of global map data and coordinate systems, including vector, raster, satellite, remote sensing image, etc., and supports WGS84 and web
All standard space and projection coordinate systems such as Mercator, CGCS2000, Xian80 and Beijing 54 can be transformed and adjusted according to the production environment.

Support for third party applications

Support three party system docking, such as SCADA data acquisition software, automatic chart generation software, BI data analysis software. Pyramidm
ap provides standard API interface, integrates map and production data, and forms rich thematic maps through data analysis and summary.
