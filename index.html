06：How to do CS transformation and XY adjustment?

How-to-do-CS-transformation-and-XY-adjustment

In the geographic system, there are two kinds of coordinate systems, three-dimensional geographic coordinate system and two-dimensional plane coordinate system. The former’s perspective is like overlooking the whole earth on the space orbital station, and its vision is located in three-dimensional space. As shown in the figure below:

The latter is a plane map, like positioning the field of vision to a local plane. As shown in the figure below:

The three-dimensional coordinate system is also known as the spatial geographic coordinate system, such as WGS84, EPSG Code: 4326. It is located by longitude and latitude, and expressed by the measuring angle from the point on the earth’s surface to the earth’s center. The datum planes for longitude and latitude calculation are the primary meridian and equator of Greenwich Observatory, UK. As shown in the figure below:

Two dimensional plane map uses plane coordinate system to locate, which is called projection coordinate system, also called plane rectangular coordinate system. The transformation from 3D coordinate system to 2D rectangular coordinate system is called projection. As shown in the figure below:

In this process, there will be deformation and error, and the value of deformation and error will gradually enlarge from the equator to the poles. Therefore, in the transformation of geographical coordinate system, we need to correct the converted values. Suppose you have a grid layer in WGS_ 1984_ Web_ Mercato, EPSG = 3857, and has been published and displayed on the web. As shown in the figure below.

Suppose you also have a device layer, the original coordinate system is GCS_ WGS_ 1984, epsg=4326, map service has been published,the layer as shown in the following figure.

Now, you want to overlay the two layers together to show the exact location of the device point. At this time, you will find that no matter how hard you try, the device point layer cannot be displayed. Because the coordinate systems of the two layers are different, they cannot be displayed in the same map viewer.

Next, we will transform the device layer into WGS_ 1984_ Web_ Mercato projection coordinate system. You will find that the two layers can be superimposed and displayed, as shown in the figure below (the grid line in the figure represents the auxiliary effect of projection conversion, which is not displayed in the real scene).

After stacking, you will find that there is a deviation between the device and the actual position. This deviation is caused by the projection deformation during the transformation from GCS to PCs. Next, we eliminate the position deviation.

Through the position correction tool of pyramidmap, the overall position correction of the layer can be realized, and the accurate position display can be realized finally. The final position is shown in the figure below.

PyramidMap Javascript API Application scenarios

PyramidMap Javascript API►05：Query and Identify

PyramidMap Javascript API query and identify on map

The map query function of PyramidMap supports various map platform interfaces and map resources, such as ArcGIS and its online map resources and local hosted map data, GeoServer hosted map, Google map, OSM, Mapbox, Tiantu, Gaode / Baidu map and other online resources. At the same time, it supports attribute data query and spatial geographic location query, and supports large amount of concurrent requests. Whether it is online map or offline map, it can accurately locate and integrate with the user’s own business data. According to the user server performance and network load, the return value of the request response can be adjusted to ensure the smoothness and stability of the map display and the accuracy of the data.：

Coordinate system compatibility and accurate positioning

PyramidMap integrates the transformation and position calibration of different coordinate systems, including GCS spatial coordinate system and PCs projection coordinate system. It also carries out compatible processing for special coordinate definitions such as Gaode / Baidu to achieve accurate positioning.

Attribute data query

The PyramidMap map query interface supports attribute query of multi data source layers, parsing layer data structure, joint query of multi fields, accurate positioning, and synchronous display of detailed information of positioning objects.

Spatial data query

PyramidMap supports the query of layer graphic spatial data. For example, on the urban pipe network map, you can filter and query according to the pressure value and flow direction, and you can also query according to the scope of urban area or community.
